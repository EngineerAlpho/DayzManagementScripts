How Everything Connects
Script 1 vs Script 2: Use download_workshop_mod.sh for installing mods one-by-one interactively. Use batch_install_mods.sh when you have a pre-made list of mods to install in bulk.
Script 3: Use cleanup_mods_config.sh anytime you need to remove mods from your configuration without deleting the actual mod files - great for testing different mod combinations.
1/2 → 4: After downloading mods (individually or in batch), use reorder_mods.sh to set proper load order (frameworks first, then content mods).
4 → 5: Once load order is correct, use dayz_mod_xml_merger.py to merge mod XMLs into server XMLs so items spawn in-game.
5 → 6: When mods update on Workshop, run update_all_mods.sh to download new versions, then re-run the merger if XMLs changed.
Typical Flow (New Server): Create mod list → Batch install → Check load order → Merge XMLs → Test server → Update mods periodically
Typical Flow (Single Mod): Download single mod → Check load order → Merge XMLs → Test server
Key Files:
  • ~/.dayz_mod_mapping - Tracks Workshop ID to mod name (used by updater)
  • ~/lgsm/config-lgsm/dayzserver/dayzserver.cfg - Mod load order configuration
  • ~/mpmissions/*/db/*.xml - Server spawn configuration (merge target)
  • ~/serverfiles/mods/@ModName/ - Installed mod files
  • example_mod_list.txt - Template for batch installations
